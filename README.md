# A Load-Cell Alternative Weighing System Based on Dual-Channel Spatial Hall Sensor Fusion and Multi-Region Hybrid Modeling

This repository contains the data, analysis scripts, manuscript figures, and STM32 firmware artifacts for a low-cost weighing system based on dual UGN3503UA Hall-effect sensors, spatial sensor fusion, EMA filtering, live tare compensation, and a multi-region hybrid calibration model.

The prototype covers the 0-4 kg range. The manuscript reports an independent live-validation MAE of 27.1 g, RMSE of 45.0 g, and a dynamic-ramp hysteresis ratio of 9.44%.

## Repository Structure

```text
data/       Clean CSV data and cached MLP baseline arrays
analysis/   Public Python scripts for metrics and figure reproduction
figures/    Final manuscript figure assets
firmware/   STM32 Core source snapshot and key flat-review files
docs/       Data dictionary, reproduction notes, and firmware notes
```

## Data

The main data files are:

- `data/calibration_data.csv`: 41 calibration anchors with raw and stick-slip-corrected AVG ADC values.
- `data/live_validation_data.csv`: 41 independent live validation measurements used for MAE/RMSE.
- `data/hysteresis_ramp_data.csv`: continuous loading-unloading ramp samples used for the 9.44% hysteresis result.
- `data/mlp_training_data.csv`: deterministic Gaussian-augmented MLP training pairs generated from the corrected calibration anchors.
- `data/stable_plateau_samples.csv`: retained plateau samples used for diagnostic hybrid-vs-MLP comparisons.

See `docs/data_dictionary.md` for column definitions and units.

## Reproducing Results

Install the required Python packages:

```bash
python -m pip install -r requirements.txt
```

Run the primary checks:

```bash
python analysis/calibration_model.py
python analysis/validation_analysis.py
python analysis/hysteresis_analysis.py
python analysis/mlp_baseline.py
```

Generate reproducible analysis figures into `outputs_tmp/generated_figures/`:

```bash
python analysis/generate_figures.py
```

To write regenerated figures directly into `figures/`, pass `--output-dir figures`. Figure 1 image panels and Figure 2 hardware architecture are included as final manuscript assets rather than generated by Python.

The standard reproduction path above uses the cached MLP arrays in `data/mlp_baseline_cache.npz` so that the manuscript values are stable. To retrain the Edge-AI baseline from scratch and rebuild the cache, install the optional training dependencies and run:

```bash
python -m pip install -r requirements-training.txt
python analysis/train_mlp_baseline.py
```

Retraining can produce small gram-level differences across TensorFlow/Keras versions and CPU backends. Keep the checked-in cache unchanged when reproducing the exact manuscript tables and figures.

## Firmware

The firmware snapshot is from the STM32CubeIDE `Core` directory. The active `main.c` implements startup tare, live tare over the Nextion UART, 32-sample oversampling, dual-channel AVG fusion, EMA filtering with `alpha = 0.25`, the hybrid LUT/cubic calibration model, a 4 kg safety clamp, and a 500 g LED threshold.

`nn_model.h` contains the compact pure-C MLP inference artifact for the edge-AI baseline. In this source snapshot, the active `main.c` uses the hybrid model and does not call `AI_Agirlik_Tahmin`; see `docs/firmware_notes.md` before rebuilding or switching firmware variants.
